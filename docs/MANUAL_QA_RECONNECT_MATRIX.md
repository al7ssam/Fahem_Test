# مصفوفة اختبار يدوي — إعادة الربط ووضع الفرق

استخدم بعد كل إصدار حساس للشبكة. علامة ✓ يدوية.

## عام

| السيناريو | خطوات مختصرة | متوقع |
|-----------|----------------|--------|
| لوبي عام → مباراة | دخول، جاهزية، انتظار العد | بدء `game_started` ثم `match_resume_token` يُخزَّن في الجلسة |
| غرفة خاصة فردية | إنشاء/انضمام، بدء | أعضاء يخرجون من لوبي الغرفة على المقبس عند `startPrivateRoomMatch`؛ انقطاع أثناء المباراة لا يزيل عضوًا من الغرفة (المقبس غير مربوط بخريطة الغرفة أثناء اللعب) |
| فردي (solo) | بدء تعلم فردي | لا تعارض مع `resume_match` عند عدم وجود تخزين |

## إعادة الربط (grace + resume)

| السيناريو | خطوات | متوقع |
|-----------|---------|--------|
| انقطاع قصير أثناء سؤال | قطع Wi‑Fi 2–5 ث ثم إعادة | خلال ~45 ث: `resume_match` ينجح، واجهة السؤال تعود؛ لا إقصاء فوري |
| انقطاع أثناء السؤال حتى انتهاء المؤقت | قطع الشبكة ثم عدم الاستئناف حتى نفاد الوقت | الجولة تُغلق عبر `finishRound` كالمعتاد؛ لا ازدواجية غير متوقعة لـ `finishRound` |
| انقطاع طويل | قطع >45 ث | إقصاء المقعد كسابق؛ `game_over` أو استمرار المباراة حسب القواعد |
| `rate_limited` على resume | إرسال عدة `resume_match` بسرعة من نفس المقبس | بعد ~25 طلباً/دقيقة يردّ الخادم `rate_limited`؛ العميل يعيد المحاولة مع backoff دون مسح التخزين |
| تبويبان / آخر مقبس | فتح لعبتين بنفس الحساب أو نفس الرمز | المقبس الأحدث يفوز؛ القديم يُفصل |
| رمز خاطئ | تلاعب بـ sessionStorage | `resume_match` يرفض `bad_token` |
| بعد انتهاء المباراة | محاولة resume | `no_match` أو `cannot_resume`؛ مسح التخزين على العميل عند `game_over` |

## وضع الفرق

| السيناريو | متوقع |
|-----------|--------|
| أول إجابة تُقفل الفريق | `team_answer_locked` + تعطيل الأزرار للزملاء |
| كابتن + تصويت | أرقام على الخيارات من `team_vote_update`؛ حالة قفل واضحة عند الإرسال |
| انقطاع أثناء التصويت | بعد الاستئناف تُستعاد أصوات الفريق من لقطة `teamVoteResync` عند توفرها |

## مذاكرة / درس

| السيناريو | متوقع |
|-----------|--------|
| استئناف أثناء بطاقات المذاكرة | لقطة `study` في الـ ACK تُعيد الطور والعداد قدر الإمكان |
| استئناف بين المذاكرة والسؤال | انقطاع بعد `study_phase_end` وقبل ظهور السؤال | اللقطة تحتفظ بـ `study` مع `matchPhaseHint: between`؛ العميل يعرض انتقالاً قصيراً حتى يصل السؤال |
| استئناف أثناء سؤال بعد المذاكرة | لقطة `question` تُعيد الواجهة |
| مشاهد بعد القلوب | قبول «متابعة كمشاهد» | `continue_as_spectator` مع `participantId`؛ انضمام لغرفة المباراة + لقطة + `keys_room_state` |

## مراقبة خادم

- لا تكرار غير متوقع لـ `finishRound` في السجلات.
- رفض إجابات بعد `roundClosed` كما سابقًا.
- سجلات **`[fahem_reconnect]`** لمسار الاستئناف؛ **`[fahem_private_room_metrics]`** لحجم الغرف الخاصة والمباريات النشطة (دوري).
